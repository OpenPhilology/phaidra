{% trans "User-Oriented Features" %}

  • {% trans "Move words via node circles" %}{% trans " — Provides users with a consistent area they can click to move a word, no matter if text is a giant word or a tiny punctuation mark. A click interaction rather than drag-n-drop is also more mobile-friendly." %}
  • {% trans "Zoom and pan" %}{% trans " — Ideal for building large trees which cannot fit on your monitor." %}
  • {% trans "Editor modal is draggable" %}{% trans " — This way, you can still see the word you're working on and the complete sentence while you're editing its part of speech, morphology, or relation attributes. It'll also continue popping up in the same place if you relocate it." %}
  • {% trans "Answer-checking against gold-standard tree" %}{% trans " — On a tree in edit mode, the node fills in when a user has placed it under the correct parent." %}
  • {% trans "Node's corresponding word highlights in sentence" %}{% trans " — So it's clear which instance of a word in the sentence you're working with." %}
  • {% trans "Part of Speech Legend" %}{% trans " — Handy, fixed position legend which tells users what the node colors indicate." %}

{% trans "Technical Features" %}

  • {% trans "Built on d3.js{% trans " — Popular, well-supported, standards-compliant data visualization framework.
  • {% trans "Browser Compatibility{% trans " — As a result, this parse tree editor works in Chrome, Safari, and Firefox (and hopefully, soon, a recent IE version).
  • {% trans "AMD Compliant{% trans " — You can include this parse tree editor into your page easily with RequireJS.
  • {% trans "Flexible Word IDs{% trans " — Word IDs do not have to be sequential, start from zero (including root), or even be integers. Useful for presenting automatically-shortened sentences without assigning words artifical ids.
